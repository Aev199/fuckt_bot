# Web Cabinet Redesign Roadmap

## Goal

Переделать web-кабинет из editor-first формы в viewer-first справочник с ограниченным редактированием по списку Telegram ID.

## Phase 1. Stabilization

### 1. Fix current web save flow

Сделать:

- корректное сохранение изменений
- понятные сообщения об ошибке
- понятное сообщение об успешном сохранении
- обновление detail-pane после сохранения

Результат:

- пользователь видит, что изменение реально применилось

### 2. Add editor permissions

Сделать:

- `WEB_EDITOR_TELEGRAM_IDS` в конфиге
- backend helper `can_edit`
- read-only поведение для пользователей без прав

Результат:

- просмотр доступен, редактирование ограничено

## Phase 2. Data Structure

### 3. Enforce one-level category nesting

Сделать:

- валидацию `parent_id`
- запрет на вложенность глубже одного уровня
- удобное дерево категорий в API

Результат:

- простая и стабильная структура `категория -> подкатегория`

## Phase 3. Frontend Redesign

### 4. Rebuild layout to 3 columns

Сделать:

- левую панель дерева категорий
- центральную панель списка материалов
- правую панель просмотра материала

Результат:

- web становится удобным для чтения и навигации

### 5. Split view and edit modes

Сделать:

- default mode: view
- edit mode по кнопке
- скрытие edit actions для read-only пользователей

Результат:

- интерфейс перестаёт выглядеть как форма по умолчанию

### 6. Improve list browsing

Сделать:

- поиск
- фильтры `Все / Избранное / С фото / Последние`
- краткие превью карточек
- показ категории и подкатегории

Результат:

- список становится полезным справочником

## Phase 4. Category UX

### 7. Category tree and subcategory CRUD

Сделать:

- создание категории
- создание подкатегории
- отображение дерева
- фильтрацию материалов по выбранной ветке

Результат:

- структура становится удобной без перегруза

## Phase 5. Detail Pane

### 8. Better material reading experience

Сделать:

- выразительный detail view
- нормальный блок фотографий
- быстрые действия: избранное, открыть источник

Результат:

- material detail становится полноценной справочной карточкой

## Suggested Implementation Order

1. починить сохранение в текущем web
2. добавить `WEB_EDITOR_TELEGRAM_IDS`
3. расширить auth/me данными о правах
4. сделать category tree API
5. перестроить frontend в layout `tree + list + detail`
6. включить read-only / edit mode
7. добавить подкатегории
8. отполировать визуал и сообщения

## Acceptance Checklist

- изменение материала через web реально сохраняется
- пользователь без прав не может редактировать через API и UI
- дерево категорий отображается корректно
- подкатегории работают и не уходят глубже одного уровня
- список и detail-pane удобны для ежедневного использования
